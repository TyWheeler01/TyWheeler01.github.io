Artifact 1: Goal Weight Tracker App

Category: Software design and engineering

Narrative for Artifact 1

This Android application allows users to log their weight, set goal weights and receive notifications according the the milestones and goals they set. The original version of the app, built in Java using SQLite, lacked modern architecture, secure logins, and visual clarity. This enhanced version addresses these issues through a few significant updates.

Enhancements completed:

  • Implemented Firebase Authentication with email verification and two-factor authentication.
  • Refactored app architecture using MVVM (Model-View-ViewModel) pattern.
  • Redesigned the user interface using Material Design components.
  • Validated weight and date inputs using regex and SimpleDateFormat.
  • Added back button navigation with consistent behavior across all screens.

Artifact 2: Treasure Hunt Agent

Category: Algorithms and data structures

Narrative for Artifact 2

This project involved creating a reinforcement learning agent for a treasure hunting game environment. The original version of the agent used basic Q-learning with limited learning stability. For this enhanced artifact I focused on improving the agent’s performance and visualizing its learning behavior.

Enhancements completed:

  • Implemented a Double DQN agent to reduce Q-value overestimation and improve overall training stability.
  • Introduced heatmap visualizations of Q-values and a line graph to better understand policy development during training.
  • Refactored the training loop for cleaner separation of logic and improved modularity.

Artifact 3: Grazioso Salvare Web Application

Category: Databases

Narrative for Artifact 3

The original Grazioso Salvare application was built as a full-stack web app to help a rescue organization manage a large database of dogs available. My enhancements focused on restructuring the backend to improve security, scalability, and maintainability of the database and application.

Enhancements completed:

  • Implemented a secure RESTful Flask API to handle the database interactions.
  • Added role based access control to ensure only admin users could perform certain operations such as deleting records.
  • Separated database logic from the UI by decoupling the Dash frontend from direct database calls.
  • Improved modularity and maintainability by organizing code into MVC style components.
